# minibo
個人利用を前提としたシンプルな家計簿Webアプリです。  
学習意図も兼ねており、使用機会のなかった技術スタックで開発しています。

## 技術スタック
**バックエンド**
|分類|技術|選定理由|
|---|---|---|
|言語|Go/Gin/Gorm|堅牢性と学習コストの低さの両立|
|DB|PostgreSQL|経験のあるミドルウェアでの開発効率の向上

**フロントエンド**
|分類|技術|選定理由|
|---|---|---|
|言語|Svlete5/TypeScript|平易な記法による学習コストの低さ|
|ランタイム|Bun|高速な動作による開発効率の向上|

## アーキテクチャ
小規模なシステムになるため過剰な切り分けは行わず、シンプルな**レイヤードアーキテクチャ**を基本としています。

|層|主な責務|
|---|---|
|Handler|HTTPリクエストのハンドリング|
|Service|ビジネスロジックの実行|
|Repository|データアクセス処理|

**設計の工夫**
- 単純なレイヤードの依存関係にならないよう、各層はinterfaceに依存するようになっています。
- 最低限のテスト、CIパイプラインによって品質を担保しています。  
- 機能数の増加に合わせて、都度テストコードの充実とアーキテクチャの変更を柔軟に検討していく予定です。
